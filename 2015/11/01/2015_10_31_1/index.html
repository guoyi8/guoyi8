python3.4.3 + OpenCV3.0.0 + dlib-18.18 on windows编译

发表于 2015-11-01   |   分类于 Python   |

引言

项目起源于github上一个使用dlib、openCV，以Python语言完成的换脸程序
Switching Eds: Face swapping with Python, dlib, and OpenCV
目前网上提供的此类解决方案大多为Python2.7 + OpenCV 2.2 ，在此记录博主的编译过程，此前并未过多涉猎Python，并且此文为事后整理，如描述有误或疏漏，请大家见谅。

下载&安装


1、从Python官网下载安装Python3.4.3
正常安装即可，注意选择添加环境变量那个选项，不赘述，正常安装即可。
2、从OpenCV官网下载目前最新版的OpenCV3.0.0，国内访问OpenCV有时候会非常卡，可以尝试github地址或其他方式下载，下载完毕后打开选择目录解压即可。
3、下载安装NumPy、SciPy
请直接选择链接下载，注明为Python3.4大版本。直接安装，会默认自动寻找Python路径。
4、下载CMake工具
此工具直接下载ZIP版本解压即可，无需安装，博主使用的目前最新版本3.4.0-rc2

编译

OpenCV编译

1、打开解压后的 cmake-3.4.0-rc2-win32-x86\bin\下的cmake-gui.exe
2、将openCV解压后的路径，参照图示填入，选择Configure，弹出框中选择VS2012 vc11(博主本机使用的VS2012，请根据实际使用选择)，但不建议使用x64即64位版本。
3、如无错误产生，点击Generate即可生成代码工程
4、用VS2012打开已生成的OpenCV工程，对工程进行Release编译生成即可。
5、将F:\Code\Python\opencv-3.0.0\opencv\build\文件夹下如图所示下面几个文件夹拷贝到个人喜好的OpenCV文件夹下，并且将此文件夹路径添加到环境变量path下，这即为OpenCV我们目前所需所有文件。在\lib\Release文件夹下找到cv2.pyd文件，拷贝到Python路径Python34\Lib\site-packages\文件夹下。此时可以对OpenCV工程进行清理，节省硬盘空间

Boost编译

1、在管理员命令行模式下，执行boost_1_59_0\bootstrap.bat，会生成b2.exe，bjam.exe
2、执行bjam.exe，此命令会根据本机编译器环境，生成库文件
3、注意执行下面命令，很重要
b2 -a --with-python address-model=32 toolset=msvc runtime-link=static
指定编译Python所需的boost库文件
注意一定统一使用32位，要么可能会产生错误

dlib编译

博主开始尝试了很久直接使用CMake编译dlib，但始终在Boost_PYTHON_LIBRARY_RELEASE/DEBUG选项上，选择任意一个可能的文件，都最终产生错误编译不过，后来使用命令方式终于成功
1、首先在上述Boost编译时使用的b2.exe那条命令所编译产生文件为基础
2、进入dlib-18.18文件夹下，管理员命令行下执行
python setup.py install --cmake x:\cmake-3.4.0-rc2-win32-x86\bin\cmake.exe
–cmake后面为本地cmake.exe路径， 博主此处未补充完整，请根据各自环境填写
3、在F:\Code\Python\dlib-18.18\tools\python\build\Release\文件夹下，将生成好的dlib.pyd文件拷贝到Python路径Python34\Lib\site-packages\文件夹下，即大功告成。
